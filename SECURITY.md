# Security Policy

## Supported Versions

We provide security fixes exclusively for the latest major version.

| Version | Supported |
| --- | --- |
| 2.x | Yes |
| < 2.0 | No |

## Reporting a Vulnerability

If GitHub private vulnerability reporting is active on this repository, please utilize it to submit your report securely.

If private reporting is unavailable, open a regular GitHub issue with a concise, non-sensitive description. Request a private disclosure channel from a maintainers before sharing exploits, secrets, or payloads.

Please include:

* The affected package version.
* A clear description of the potential impact.
* Safe steps to reproduce the issue, if they can be shared securely.
* Any known temporary workarounds.

We review valid reports promptly to coordinate fixes and issue security releases.
